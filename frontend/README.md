# Frontend — Коммунальные платежи

Веб-интерфейс для системы учёта коммунальных платежей.

## Технологии

- Чистый JavaScript (ES6+, модули)
- Чистый CSS (Flexbox, Grid, CSS Variables)
- Google Fonts: DM Sans + JetBrains Mono
- Без React/Vue/Angular — нативный DOM API

## Структура

```
frontend/
├── index.html          — Главная: таблица со статусами, фильтр, статистика
├── account.html        — Детали счёта: платежи, показания, формы
├── css/
│   └── styles.css      — Все стили (Industrial Utilitarian дизайн-система)
└── js/
    ├── api.js          — Обёртка над сгенерированным typescript-fetch клиентом
    ├── ui.js           — Переиспользуемые UI-компоненты (el, модалки, бейджи, валидация)
    ├── main.js         — Логика Dashboard-страницы
    └── account-page.js — Логика страницы деталей лицевого счёта
```

## Сгенерированный API-клиент

Frontend использует клиент, сгенерированный из OpenAPI-спецификации бэкенда:

```bash
# 1. Запустить бэкенд
mvn spring-boot:run

# 2. Сгенерировать клиент
openapi-generator-cli generate \
  -i http://localhost:8080/v3/api-docs \
  -g typescript-fetch \
  -o frontend/js/generated-client

# 3. Скомпилировать TypeScript → JavaScript
cd frontend/js/generated-client
tsc --outDir . --declaration false
```

### Маппинг API-классов

| @Tag (контроллер)      | Сгенерированный класс     | Методы                                |
|------------------------|---------------------------|---------------------------------------|
| Услуги                 | `УслугиApi`               | `getAll()`                            |
| Лицевые счета          | `ЛицевыеСчетаApi`         | `create()`, `getById()`, `getWithStatus()` |
| Платежи                | `ПлатежиApi`              | `create1()`, `getByAccount()`         |
| Показания счётчиков    | `ПоказанияСчётчиковApi`   | `create2()`, `getByAccount1()`        |

> Имена методов соответствуют operationId из springdoc-openapi. Суффиксы `_1`, `_2`
> добавлены автоматически для разрешения конфликтов имён между контроллерами.

## Запуск

Для локальной разработки можно использовать любой статический сервер:

```bash
# Вариант 1: npx
npx serve frontend

# Вариант 2: Python
cd frontend && python -m http.server 3000
```

Бэкенд должен быть запущен на `http://localhost:8080` (или настройте `window.API_BASE_URL`).

## Функциональность

### Главная (index.html)
- Карточки статистики: всего / оплачено / не оплачено / просрочено
- Фильтрация по статусу (pill-кнопки)
- Кликабельная таблица с переходом на детали

### Детали счёта (account.html)
- Breadcrumb-навигация
- Информация о счёте (услуга, номер, наличие счётчика)
- История платежей (таблица, сортировка по дате DESC)
- История показаний (для услуг со счётчиком)
- Модальные формы добавления с клиентской валидацией
- Отображение серверных ошибок из ErrorResponse
- Спиннеры при загрузке и отправке
